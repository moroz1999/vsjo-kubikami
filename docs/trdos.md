# TR-DOS Disk Loading

## Disk Variants and File Order

All five languages are exported as regular TRD and GS TRD images.

| Variant | Physical file order |
|---|---|
| Regular | `boot.B`, `LOADER.C`, `LOADING.C`, `KUBIKAMI.C` |
| General Sound | `boot.B`, `LOADER.C`, `LOADING.C`, `GSDATA.C`, `KUBIKAMI.C` |

The order is part of the loader protocol. BASIC locates only `LOADER.C` by name. All later data is read by physical track and sector without directory lookup.

`emptytrd` creates the image, and consecutive `savetrd` directives place the files in the required order. Each file occupies `ceil(length/256)` sectors, including padding in its final sector.

## Disk Geometry and Cursor

- One sector is `256` bytes.
- One track contains `16` sectors.
- TR-DOS variable `#5CF4` stores the current sequential read position.
- When loaded with `ld de,(#5CF4)`, `D` is the track and `E` is the sector `0..15`.

The named BASIC load of `LOADER.C` leaves `#5CF4` at the first sector after that file. Every successful raw read advances it, allowing the loader to consume the remaining files sequentially.

## `#3D13` Read Contract

`trd_loader_read_sectors` calls TR-DOS routine `#3D13` with:

| Register | Value |
|---|---|
| `HL` | Destination address |
| `DE` | Track in `D`, sector in `E` |
| `B` | Sector count |
| `C` | Function `5`, read sectors |
| `IY` | System variables at `#5C3A` |

Before the call, the loader selects IM 1. After the call it executes `di`, because TR-DOS may alter interrupt state. The game later builds its IM 2 table and selects IM 2 itself.

The sector count is one byte. The loading screen and whole game must each fit in at most `255` sectors; assembler assertions enforce this limit.

## Sector Counts

`LOADING.C` is `6912` bytes, exactly `27` sectors.

The game count is calculated in assembly:

```asm
(@end-@start+255)/256
```

The current localized game images occupy `136` sectors. The count must be recalculated from labels, not copied as a fixed value.

GS block sizes and the total GS skip are generated by `gs/prepare.php`. The current `GSDATA.C` is `227` sectors, represented as:

- `gs_data_skip_tracks = 14`
- `gs_data_skip_sectors = 3`

When GS is absent, the loader adds the sector remainder to `E`, converts overflow past sector `15` into a track increment, adds the generated track count to `D`, and writes the result back to `#5CF4`. This must land on the first sector of `KUBIKAMI.C`.

## Loader Memory Map

The disk loader must survive while `KUBIKAMI.C` is loaded over the main game range.

| Range/address | Use |
|---|---|
| `#4000..#5AFF` | Loading screen |
| `#5B00` | GS-detected flag |
| `#5CCB...` | Runtime BASIC program and workspace |
| `#6000..#E8FF` | Localized game image |
| `#8000` | Temporary packed GS block before the game is loaded |
| `#C000` | Temporary unpacked GS block before the game is loaded |
| `#F000..#F02E` | Regular loader |
| `#F000..#F1FA` | GS loader |
| `#FDFD..#FF00` | Future game IM 2 dispatch and vector area |
| `#FFFF` downward | Temporary loader stack |

The `#8000` and `#C000` buffers deliberately overlap the future game image: GS data is processed first, and `KUBIKAMI.C` overwrites those buffers afterward. The largest unpacked block is `10000` bytes, ending below `#F000`.

The loader stack may temporarily use the future IM 2 area because the vector table does not exist yet. On game entry, `startup` immediately changes SP to `#5FFF`; `interrupts.init` builds the IM 2 structures later.

The required invariants are:

- `end <= trd_loader`.
- `trd_loader_end <= interrupts.reserved_start`.
- The packed and unpacked GS buffers must not reach `#F000`.
- The loader must not be placed inside the range overwritten by the game read.

## Build-Time Data Staging

`release.a80` first saves `LOADING.C` from `#4000`. For a GS image, it then reuses the same `6912`-byte assembly-time memory area to save `samples.bin` in sector-aligned chunks. The first chunk creates `GSDATA.C`; later chunks use `savetrd` append mode, so the physical file stays contiguous before `KUBIKAMI.C`. Append mode updates the catalog sector count; the loader ignores the byte-length field and reads the generated block counts directly. The total GS stream is limited to the TR-DOS catalog maximum of `255` sectors.

See [TRD BASIC Loader](basic_loader.md) for the bootstrap and [General Sound](general_sound.md) for `GSDATA.C`.
