# ТЗ для ИИ-агента

## Задача

Нужно реализовать логику босса для ZX Spectrum на ассемблере.

Босс состоит из контрольных точек с целочисленными координатами.  
Точки соединяются линиями.  
Логика должна быть дешёвой по вычислениям и пригодной для ZX Spectrum.

## Состав модели

### Статические точки
- `W1`
- `W2`

Это точки на стене. Они не двигаются.

### Динамические точки
- `A`
- `B`
- `C`

### Роли точек
- `C` это голова
- `B` связана с `C`
- `A` находится ближе к стене

## Принцип работы

Каждый кадр нужно:
1. пересчитать координаты динамических точек
2. после этого нарисовать линии между точками

Голова `C` должна двигаться за героем.

Для головы используются смещения от дефолтного положения:
- `dCx`
- `dCy`

Есть дефолтные координаты:
- `sBx`
- `sBy`
- `sCx`
- `sCy`

Для точки `B` используется формула:

`Bx = sBx + dBx + dCx / 2`

Для точки `C` используется такой же принцип, но относительно `B`.

Все вычисления должны быть целочисленными.

## Ограничения

- ZX Spectrum
- ассемблер
- только целочисленные вычисления
- без тяжёлой физики
- без лишних сущностей
- без лишней логики, которая не указана в задаче
- названия процедур не нужны

## Что нужно сделать

1. Реализовать хранение координат статических и динамических точек.
2. Реализовать пересчёт координат динамических точек каждый кадр.
3. Реализовать движение головы `C` за героем через смещения от дефолтного положения.
4. Реализовать зависимость `B` от `C` по заданной формуле.
5. Реализовать такой же принцип расчёта `C` относительно `B`.
6. Реализовать отрисовку линий между точками.

## Что нужно запросить до реализации

Перед реализацией обязательно запросить конкретные исходные данные:

1. Координаты `W1`
2. Координаты `W2`
3. Координаты `A`
4. Координаты `B`
5. Координаты `C`
6. Какие именно линии нужно рисовать между точками
7. В какой системе координат это хранится
8. Где хранятся координаты героя
9. Какой формат данных нужен для координат
